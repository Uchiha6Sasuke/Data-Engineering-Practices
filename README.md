# Data-Engineering-Practices
Репозиторий, содержащий ноутбуки с проектами по аналитике и исследованию данных

1. Model_Monitoring_Dashboard.ipynb содержит создания дэшборда мониторинга модели машинного обучения.
     1. Загрузка датасета с мошенническими транзакциями по кредитным картам (kaggle);

     2. EDA;

     3. Подготовка данных - разделение на train/test, стандартизация признаков;

     4. Обучение модели - логистическая регрессия с учетом дисбаланса классов;

     5. Оценка метрик;
   
     6. Мониторинг модели - анализ дрейфа признаков и предсказаний;

Модель обучена предсказывать мошеннические операции, анализируется стабильность предсказаний модели во времени. Проверяется дрейф распределений признаков между обучающими и новыми данными. Используются метрики, устойчивые к дисбалансу классов.

2. Data_Quality_&_Profiling.ipynb содержит комплексный анализ качества данных и их профилирование.
     1. Загрузка и подготовка датасета с мировыми показателями развития (kaggle);
   
     2. Всесторонняя проверка качества данных: пропуски, дубликаты, типы данных;
   
     3. Создание небольшой функции для проверки качества данных по нескольким критериям;
   
     4. Визуализация статистики (matplotlib);
   
     5. Анализ аномалий и выбросов данных;

Ноутбук служит для анализа мировых тенденций развития и сравнительных исследований между странами.

3. Data_metrics_DPA.ipynb содержит анализ данных с помощью функций pyspark и реализацию Data Pipeline Architecture.
      1. Загрузка датасета с информацией о пассажирах Титаника (kaggle);
         
      2. Проверка качества данных по метрикам Completeness, Uniqueness, Consistency;
         
      3. Аналитические вычисления (группировка, ранжирование);
         
      4. Реализация Data Pipeline Architecture;
  
Ноутбук демонстрирует полный цикл работы с данными: от загрузки и проверки качества до feature engineering.
